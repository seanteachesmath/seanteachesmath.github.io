The worst possible initial guess would be at one of the endpoints, so we will start there (this maximizes the term \(\max(p_0-a, b - p_0)\text{,}\) which in this case we actually want to do in order to generate a conservative bound). Taking \(D\) as the desired accuracy (i.e., an accuracy within \(10^{-D}\)), this gives,

\begin{equation*} \begin{aligned} k^n \max(p_0-a, b - p_0) & < 10^{-D}\\ (0.551)^n \left(\frac{2}{3}\right) & < 10^{-D}\\ (0.551)^n & < \left(\frac{3}{2}\right)10^{-D}\\ n \log(0.551) & < \log\left(\frac{3}{2}\right) - D\\ n & > \frac{\log\left(\frac{3}{2}\right) - D}{\log(0.551)}\end{aligned} \end{equation*}

In the last line, the inequality has been reversed since we are dividing by a negative. With \(D=4\) this gives \(n > 14.77277\) which requires \(N=15\) steps.
